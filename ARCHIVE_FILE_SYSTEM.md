## نظام إدارة الملفات المُحسّن للأرشيف

### المقدمة
تم تطوير نظام منظم وموحد لحفظ واسترجاع المستندات والملفات الخاصة بالمذكرات والأرشيف.

### المشكلة التي تم حلها
- ✅ حفظ الملفات بطريقة منظمة في مجلد مخصص
- ✅ تسمية موحدة للملفات: (موضوع المذكرة - رقم المذكرة - الطابع الزمني)
- ✅ إدارة مركزية لجميع عمليات الملفات
- ✅ استرجاع سهل من الأرشيف باستخدام البيانات الوصفية

### بنية التخزين

```
App Documents Directory (متغير حسب النظام)
└── mudiri_archive/                    # المجلد الرئيسي للأرشيف
    ├── تقرير شهري-2025-1700000000.pdf
    ├── مذكرة إدارية-101-1700000100.pdf
    ├── عقد توظيف-خ-201-1700000200.docx
    └── ...
```

### مكونات النظام

#### 1. **FileStorageService** (`lib/core/services/file_storage_service.dart`)
خدمة مركزية لإدارة جميع عمليات الملفات:

**الدوال الرئيسية:**
- `getArchiveDirectory()` - الحصول على مجلد الأرشيف
- `generateFileName()` - إنشاء أسماء ملفات موحدة
- `saveFile()` - حفظ ملف في الأرشيف
- `getFile()` - استرجاع ملف من الأرشيف
- `deleteFile()` - حذف ملف من الأرشيف
- `listAllFiles()` - سرد جميع الملفات
- `getFileSize()` - الحصول على حجم الملف

#### 2. **ArchiveRepository** (`lib/features/archive/domain/archive_repository.dart`)
تم تحديثه ليستخدم FileStorageService:

```dart
// عند إنشاء سجل أرشيف
await repository.createArchiveRecord(
  title: 'تقرير شهري',
  referenceNumber: '2025/001',
  localFilePath: '/path/to/document.pdf',
  // ... باقي البيانات
);
```

يتم حفظ الملف بشكل تلقائي مع:
- اسم منظم: `تقرير شهري-2025/001-1700000000.pdf`
- مسار محفوظ في قاعدة البيانات

#### 3. **ArchiveFileHelper** (`lib/features/archive/presentation/archive_file_helper.dart`)
أداة مساعدة للتعامل مع ملفات الأرشيف في واجهة المستخدم:

```dart
// فتح ملف من الأرشيف
ArchiveFileHelper.openFile(
  filePath: record.localFilePath,
  onError: (error) => showErrorMessage(error),
);

// عرض معلومات الملف
ArchiveFileHelper.buildFileInfoWidget(
  filePath: record.localFilePath,
  context: context,
);

// الحصول على حجم الملف
final size = await ArchiveFileHelper.getFileSizeDisplay(record.localFilePath);
```

### معايير التسمية الموحدة

**الصيغة:**
```
{موضوع المذكرة}-{رقم المذكرة}-{الطابع الزمني}.{الامتداد}
```

**أمثلة:**
- `مذكرة أدارية-101-1700000000.pdf`
- `تقرير سنوي-2025-1700000100.xlsx`
- `عقد توظيف-خ-201-1700000200.docx`

**الميزات:**
- الأحرف غير المسموحة (`\ / : * ? " < > |`) يتم استبدالها بـ `-`
- الفراغات المتعددة يتم تحويلها إلى فاصل واحد `-`
- الطابع الزمني يضمن عدم تضارب الأسماء

### سير العملية

#### إنشاء سجل أرشيف جديد:
1. ✅ المستخدم يملأ نموذج المذكرة
2. ✅ يختار أو يلتقط ملف/مستند
3. ✅ عند الحفظ:
   - يتم إنشاء اسم موحد للملف
   - يتم حفظ الملف في `mudiri_archive/`
   - يتم حفظ المسار الكامل في قاعدة البيانات
   - يتم تسجيل العملية في السجلات

#### استرجاع سجل من الأرشيف:
1. ✅ يتم عرض قائمة المذكرات المؤرشفة
2. ✅ عند اختيار مذكرة:
   - يتم استرجاع مسار الملف من قاعدة البيانات
   - يتم التحقق من وجود الملف
   - يتم فتح الملف تلقائياً (باستخدام التطبيق الافتراضي)

#### حذف سجل من الأرشيف:
1. ✅ يتم حذف الملف الفعلي من `mudiri_archive/`
2. ✅ يتم حذف السجل من قاعدة البيانات (soft delete)
3. ✅ يتم تسجيل العملية في السجلات

### مثال استخدام عملي

#### حفظ مذكرة مع ملف:
```dart
final repository = ref.read(archiveRepositoryProvider);

await repository.createArchiveRecord(
  title: 'تقرير الأداء الشهري',
  referenceNumber: '2025/01/001',
  documentDate: '2025-01-15',
  hijriDate: '15 رجب 1446',
  directedEntity: 'قسم الموارد البشرية',
  category: 'تقارير',
  localFilePath: '/temp/performance_report.pdf',
  tags: 'أداء,تقرير,شهري',
  notes: 'تقرير الأداء الشهري للموظفين',
  isConfidential: false,
);
```

**النتيجة:**
- الملف يتم حفظه بـ: `mudiri_archive/تقرير الأداء الشهري-2025/01/001-1700000000.pdf`
- السجل يُحفظ في قاعدة البيانات مع مسار الملف

#### استرجاع وفتح الملف:
```dart
void _openArchiveFile(ArchiveData record) {
  ArchiveFileHelper.openFile(
    filePath: record.localFilePath,
    onError: (error) {
      ScaffoldMessenger.of(context).showSnackBar(
        SnackBar(content: Text(error)),
      );
    },
  );
}
```
